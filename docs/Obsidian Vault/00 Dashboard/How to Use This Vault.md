# How to Use This Vault Professionally

## Daily Workflow

1. **Start here** → [[00 Dashboard/Dashboard]]
2. Check open tasks in [[06 Tasks & Progress/Spring Jam 26]]
3. Work on a system → open its note in `03 Systems/`
4. Capture quick thoughts in [[Inbox/Inbox]]
5. At end of session → move Inbox items to proper notes + update links

## Naming Conventions

- Systems: `Player Controller.md`
- Design docs: Descriptive titles
- MOCs always end with "MOC"
- Templates live in `Templates/` folder

## Linking Rules

- Every system note links back to the Systems MOC
- Code files are mentioned with full relative path
- Use wikilinks aggressively (`[[Note Name]]`)
- Always use full paths for notes in subfolders (e.g. `[[03 Systems/Player Controller]]`)

## Best Practices

- Keep notes scannable (short paragraphs + bullet points)
- Use YAML frontmatter for status and tags
- When adding new features, create both a Design note and a System note
